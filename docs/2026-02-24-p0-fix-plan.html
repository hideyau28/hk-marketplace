## Issue 1: Homepage 大片空白 + 底部疑似重複

### 問題描述
Homepage 中段有巨大黑色空白區域（約佔 40% viewport），底部出現類似 hero 嘅 section。

### Root Cause
- **File:** `components/marketing/LandingPage.tsx`
- **Line 300-301:** 多個 section 用 `.scroll-reveal` class（opacity: 0, translateY: 24px）
- **Line 671-716:** `FINAL CTA` section 嘅內容（深色背景 + 大 headline + 橙色 CTA button）同 Hero section 視覺非常相似，構成「重複」觀感
- **空白成因：** `.scroll-reveal` sections 需要 JS scroll event 觸發 `.scroll-visible` 先會出現。Puppeteer screenshot 唔一定觸發所有 scroll event，導致中間 sections 保持 opacity: 0 — 呈現為大片空白。真實用戶使用瀏覽器時，如果 JS 載入慢或 IntersectionObserver 未 fire，一樣會出現短暫空白閃爍
- **注意：** FINAL CTA (L671-716) 係 intentional 嘅 closing CTA，唔係 bug duplicate。但視覺上同 hero 太似，令截圖誤判為 duplicate

### 修復方案
1. 將 `.scroll-reveal` 嘅初始 opacity 改為 1（`opacity: 0` → `opacity: 1`），純用 animation 做 reveal effect，避免 content invisible fallback
2. 或者加 `<noscript>` / CSS-only fallback: `.scroll-reveal { opacity: 1; }` 作為 no-JS/slow-JS fallback
3. FINAL CTA section 保留但調整視覺差異化：改用唔同 headline size 或加 testimonial 區分

### 影響範圍
- `/` (Homepage landing page only)
- 所有 locale

### 風險評估: Low
- 純 CSS 改動
- 唔影響功能

### PR: 1 PR

## Issue 3: 品牌色調分裂（3 頁 3 種風格）

### 問題描述
Landing page 用黑底，Pricing page 用白底，Store pages 用暖米色。三個頁面三種色調，品牌 identity 分裂。

### Root Cause
- **File:** `app/globals.css` L5-35 — 定義咗 CSS variables（`--tmpl-accent`, `--primary`, olive palette 等）
- **File:** `components/marketing/LandingPage.tsx` L295 — 整個 component 用 inline `style={{ color: "#1A1A1A" }}`，hardcoded `#0D0D0D`, `#FF9500`, `#111` 等
- **File:** `components/marketing/PricingPage.tsx` L614-620, L941 — 用 inline `background: "#FAFAFA"`, `#1A1A1A`, `#0A0A0A`
- **File:** `components/marketing/LandingPage.tsx` L296-318 — 所有 CSS 都寫喺 `<style>` tag 入面，冇用 globals.css 嘅 variables
- **核心問題：** Marketing pages 完全繞過 design token system，所有顏色 hardcoded inline

### 修復方案
1. 喺 `globals.css` 加入 marketing color tokens：`--mkt-bg`, `--mkt-surface`, `--mkt-text`, `--mkt-accent` (=#FF9500)
2. 逐步將 LandingPage.tsx 同 PricingPage.tsx 嘅 inline color 替換為 CSS variable references
3. 長期：統一 dark marketing theme — Landing + Pricing 共用同一套色盤
4. **注意：** 呢個係最大嘅改動，建議分兩步：
   - Step A: 抽取 color variables（唔改視覺）
   - Step B: 統一 Landing/Pricing 色盤

### 影響範圍
- `/` (Landing)

- `app/globals.css`
- 唔影響 store pages（store pages 已經用 `--tmpl-accent`）

### 風險評估: Medium
- 涉及兩個大 file（LandingPage 793 行, PricingPage 1411 行）
- Inline style 替換容易出 regression
- 建議分階段做

### PR: 2 PRs

## Issue 6: 產品圖 aspect ratio 失控

### 問題描述
部分店鋪（nichiyori, greenday）嘅產品圖只佔 card 下半部分，上半截大片空白。

### Root Cause
- **File:** `components/biolink/BioProductCard.tsx` L68-69
  ```
  className="relative aspect-square overflow-hidden cursor-pointer"
  ```
- **Line 77-87:** Image 用 `fill` + `object-cover`，理論上應該填滿
- **Image component:** Next.js `<Image fill className="object-cover" />`
- **實際問題：** 圖片本身 aspect ratio 唔係 1:1（例如 portrait 或 landscape），而 `object-cover` 應該正確裁切。如果出現空白，可能係：
  1. 圖片 URL 實際返回嘅圖片有大量 transparent/white padding
  2. 或者 Next.js Image loader 嘅 `sizes` hint 導致載入錯誤 resolution
  3. 或者圖片仲未載入完成時嘅 placeholder 狀態

### 修復方案
1. 確認 `object-cover` 已經正確套用（現有 code 看起來正確）
2. 加 `object-center` 確保裁切居中
3. 加 placeholder/blur 防止載入前空白
4. 實際測試：直接喺瀏覽器 inspect 確認係圖片內容問題定係 CSS 問題

### 影響範圍
- 所有 store pages
- `components/biolink/BioProductCard.tsx` 單一 file

### 風險評估: Low
- 可能只需要加一個 class
- 需要實際 inspect 先確認 root cause

### PR: 1 PR

1. 將 `AddToCartButton` 嘅 default className 改為用 CSS variable：
