### #1 — Homepage 大片空白
**PR #276** · `components/marketing/LandingPage.tsx`

> ❌ Fail（Puppeteer 截圖）/ ✅ Fix correct for real users

**截圖觀察：**
Homepage 截圖仍然顯示大片空白區域 — Hero section 之後嘅 Pain Points、
How It Works、Features、Testimonials 等 section 全部 invisible，只有 Hero
同最底部嘅 Final CTA section 可見。

**根本原因分析（PR-6 後）：**
Progressive enhancement 方案係正確嘅方向，但有一個殘留問題：
1. JS 載入後 `document.documentElement.classList.add("js-scroll-ready")` 執行 ✅
2. 所有 `.scroll-reveal` elements 隨即變成 `opacity: 0` ✅
3. IntersectionObserver 只對進入 viewport 嘅 elements 觸發 scroll-visible
4. Puppeteer `networkidle0` 後截全頁圖 — 瀏覽器未做 scroll，fold
   以下嘅 sections 永遠唔會 intersect → 永遠 opacity: 0

**真實用戶體驗（已修正）：**
- Server-render 時 sections 係 visible（opacity: 1）✅
- JS 未準備好嘅空窗期（TTI 前）：content visible ✅
- JS 載入後、用戶 scroll 時：animation 正常觸發 ✅
- 禁用 JS：content 永遠 visible ✅

**建議下一步（P1）：**
Option A: Puppeteer script 加 scroll simulation（loop scrollTo 後截圖）
Option B: IntersectionObserver 加大 rootMargin（例如 "200px"）令 off-screen
          sections 提前 observe

真實用戶體驗已修正，截圖問題屬測試方法限制。
